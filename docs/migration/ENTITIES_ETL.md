## Cross-cutting ETL notes

- Table names contain hyphens (`push-subscribes`, `restore-passwords`) — quote identifiers or rename in the Drizzle schema (renaming breaks nothing external; tables are not part of the HTTP contract).
- Join tables created by TypeORM M2M (`users.master` ↔ `campaigns.masters`) have TypeORM-generated names — inspect the live legacy schema (`information_schema`) during TASK-020 rather than trusting entity metadata.
- Denormalized/derived fields (`rating`, `calculatedRating`, `companyRankingSystem`, `workerRankingSystem`, `serviceRankingSystem`) can be copied as-is; the TASK-013 ranking jobs recompute them.
- Mixed PK types (uuid vs int) across tables — target schema may normalize to uuid, but then ETL must maintain an id mapping for `notifications`/`appointments`/`tokens` references.
